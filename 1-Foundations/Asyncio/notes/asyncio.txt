# ============================================================
# SYNCHRONOUS vs ASYNCHRONOUS
# ============================================================

1. Synchronous (Sync)
    - runs line by line
    - blocks execution until task finishes
    - slow for I/O-heavy tasks

    Example idea:
        request -> wait -> request -> wait

# Key Idea:
# Everything waits in a queue

# ------------------------------------------------------------

2. Asynchronous (Async)
    - runs multiple tasks concurrently
    - does NOT block execution
    - ideal for I/O tasks (API calls, DB, web scraping)

    Example idea:
        request1 + request2 + request3 run together

# Key Idea:
# Tasks can pause and resume

# ============================================================
# COROUTINES
# ============================================================

1. Coroutine
    - special async function
    - defined using async def
    - can be paused with await

    Syntax:
        async def my_func():
            await some_task()

2. Behavior
    - does NOT run immediately when called
    - returns a coroutine object

    Example:
        coro = my_func()  # not executed yet

# Key Idea:
# Coroutine = async function blueprint

# ============================================================
# TASKS
# ============================================================

1. Task
    - wraps a coroutine
    - schedules it to run in event loop
    - runs concurrently with others

    Syntax:
        task = asyncio.create_task(coroutine())

2. Behavior
    - starts execution immediately
    - managed by event loop

# Key Idea:
# Task = running coroutine

# ============================================================
# FUTURES
# ============================================================

1. Future
    - low-level async result container
    - represents a value that will exist later

2. States:
    - pending
    - done
    - cancelled

3. Usage
    - usually handled internally by asyncio
    - rarely used directly in modern code

# Key Idea:
# Future = placeholder for result

# ============================================================
# TASKGROUP (Python 3.11+)
# ============================================================

1. TaskGroup
    - modern way to manage multiple tasks
    - safer than manual task creation
    - automatically handles errors

    Syntax:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(coro1())
            tg.create_task(coro2())

2. Behavior
    - if one task fails → others are cancelled
    - structured concurrency model

# Key Idea:
# TaskGroup = safe concurrent task manager

# ============================================================
# EVENT LOOP (CORE IDEA)
# ============================================================

1. Event Loop
    - engine that runs async code
    - schedules and switches tasks

2. Responsibilities:
    - runs coroutines
    - manages tasks
    - handles I/O waiting

# Key Idea:
# Event loop = brain of asyncio

# ============================================================
# SIMPLE FLOW

# coroutine → task → event loop → execution → future(result)

# ============================================================
# REAL-WORLD USES (AI ENGINEERING)

1. API Calls (FastAPI / LLM APIs)
    - parallel requests to OpenAI, HuggingFace, etc.

2. Web Scraping
    - fetch multiple pages simultaneously

3. Database Queries
    - concurrent reads/writes

4. Microservices
    - async backend systems

# ============================================================
# MINI EXAMPLE (CONCEPTUAL)

import asyncio

async def fetch_data():
    await asyncio.sleep(1)
    return "data"

async def main():
    task1 = asyncio.create_task(fetch_data())
    task2 = asyncio.create_task(fetch_data())

    result1 = await task1
    result2 = await task2

    print(result1, result2)

# ============================================================
# SUMMARY

    - async function

2. Task
    - running coroutine

3. Future
    - result placeholder

4. TaskGroup
    - structured concurrency manager

5. Sync vs Async
    - blocking vs non-blocking execution
